# Original prompt

The repo owner is a data engineer with no finance background trying to break into the financial sector as a data engineer. They asked for a learning app to build finance domain knowledge — focused on markets/instruments and regulations/compliance — written for someone who already thinks like a data engineer (pipelines, schemas, ETL), not a finance student.

Requested learning style: structured curriculum + visual concept explorer (not a static wall of text), self-paced, no backend/repo infrastructure needed beyond the static page itself.

This resulted in a single self-contained `index.html` with 9 modules (overview, instruments, markets, participants, data, risk, compliance, pipelines, quiz), client-side progress tracking, and an applied 10-question quiz.
